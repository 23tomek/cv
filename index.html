My default response to any problem is: can I build something to solve this? Below are two examples — a consumer product I built from scratch to solve my own pain point, and a set of tools I wrote because I refused to pay for something I could make myself.

Artifact 1 — Product

Roomzy

FIND YOUR PEOPLE, FIND YOUR PLACE

As a student, I experienced first-hand how painful it is to find a flat and a compatible roommate in a new city. The process was scattered across multiple platforms, there was no trust layer, and zero focus on whether people would actually get along living together. So I built the solution myself.

Roomzy is a mobile platform that matches people by lifestyle and personality — not just listings. I designed the product, built the matching algorithm, coded the full MVP solo, and mapped out the business model and go-to-market strategy. Pre-launch stage.

Solo founder & developer Full-stack mobile app Matching algorithm Verification system In-app chat Freemium model
Login Lifestyle matching City selection Explore & profiles In-app chat

What building Roomzy taught me

  • Turning a real personal frustration into a product — from problem identification to working MVP
  • Going from zero to a functional app alone: scoping, designing, coding, iterating
  • Thinking about users, segments, and monetisation from day one (students, expats, digital nomads)
  • What founders actually go through — the discipline of building solo and iterating without external validation
Artifact 2 — Builder tools

EPUB Translator

PROBLEM: Professional book translations cost hundreds of PLN per title

Built a Python CLI tool that translates entire EPUB books using GPT-4 while preserving HTML structure, images, and paragraph formatting. Handles chunking, chapter selection, and produces publication-ready output.

# Translate chapters 1-15 of an English book to Polish
python translate_epub.py translate \
  --input book.epub --output book_pl.epub \
  --from-lang EN --to-lang PL \
  --from-chapter 1 --to-chapter 15
Python OpenAI GPT-4 BeautifulSoup ebooklib CLI

Stock Portfolio Analyser

PROBLEM: Needed portfolio analytics without paying for expensive platforms

Custom-built application for analysing my own stock portfolio — tracking positions, performance, and key metrics. Designed to give me the data I need without the cost of professional terminals.

Python Data analysis Portfolio tracking

Live Market Data System

PROBLEM: Real-time market data is locked behind Bloomberg / Refinitiv paywalls

Built a system that fetches live stock market data using free APIs — essentially a basic Bloomberg terminal for personal use. Real-time quotes, market data aggregation, and a clean interface for quick decision-making.

Python REST APIs Real-time data Market data

Why I build things

Every project above started the same way: I ran into a problem and decided to solve it myself instead of waiting for someone else or paying for an off-the-shelf solution. Roomzy started because I couldn't find a decent flatmate as a student. The EPUB translator because I wanted to read a book that didn't exist in Polish. The portfolio tools because I couldn't justify paying for Bloomberg as a 22-year-old.

This is just how I think. I see a gap, I figure out what it would take to close it, and I build. That same instinct — spotting problems, moving fast, shipping something that works — is what I'd bring to sourcing and supporting founders at Inovo.
